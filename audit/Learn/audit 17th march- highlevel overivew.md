## 🔴 Critical Issues (Fix First)

**1. Duplicate Courses on Dashboard**
The "Continue learning" section shows "Introduction to Digital Empathy for non-voice contact center agents" **twice** and "Somehow I manage" **twice**. This is a data/query bug — likely a missing `DISTINCT` or incorrect JOIN in the course-fetching logic. This will confuse learners and erode trust in the platform immediately. [sudar-learn.vercel](https://sudar-learn.vercel.app/)

**2. Day Streak is Broken**
The streak counter shows **0 days** despite the dashboard simultaneously saying "1 active day this week" and a learning time of 499 hrs. The streak logic and the activity counter are clearly out of sync — these should share the same source of truth from `learning_events`. [sudar-learn.vercel](https://sudar-learn.vercel.app/)

**3. Learning Time Contradiction**
The stats section shows **499 hrs** total learning time, but the "Your activity" section reports **374h** for "this week" and **22,440 minutes** (= 374 hrs) for the period. These numbers don't reconcile. If 374h is a single week, it's almost certainly a miscalculation (374h = 53+ hours/day). The time aggregation logic needs an audit. [sudar-learn.vercel](https://sudar-learn.vercel.app/)

**4. Greeting Rendering Bug**
The dashboard greeting renders as `"Good evening , Dhanikesh ."` with extra spaces around the comma and period. This suggests a template string issue like `Good evening {greeting}, {name}.` where `greeting` resolves empty — a small but visible polish issue. [sudar-learn.vercel](https://sudar-learn.vercel.app/)

**5. Modules Done = 0 Despite Activity**
The activity section shows **0 modules done** for the period while showing active sessions and learning time. Module completion events are either not being written to `learning_events` correctly, or the aggregation query for "modules done" is broken. [sudar-learn.vercel](https://sudar-learn.vercel.app/)
